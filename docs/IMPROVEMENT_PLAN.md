# WorkManager 실사용 개선 플랜

최종 갱신: 2026-07-10

목표: 개인 업무 관리(오늘 할 일·일정·진행도·업무 기록·성과 보고)를 매일 안정적으로 쓸 수 있는 수준으로 끌어올리고, 이후 상업 프로그램 수준의 완성도를 갖춘다.

## 현황 진단 (2026-07-10)

- 기능 뼈대(업무 Gantt·캘린더·오늘·성과·AI·감사로그·휴지통)는 갖춰져 있으나, 최근 "AI 설정" 기능이 **인코딩이 깨진 채(한글 → `?`) 커밋**되어 설정 화면 AI 섹션과 백엔드 상태 메시지가 물음표로 표시됨.
- 백엔드 테스트 67개 중 **2개 실패**: `PUT /api/settings/ai` 응답에 `provider_name`/`api_key_set` 누락(계약 위반), `ai.status()` 시그니처 변경에 못 따라간 낡은 테스트.
- 개인용 도구인데 **팀용 승인 워크플로가 강제**: 업무를 완료하면 항상 "승인 대기", 날짜를 바꾸면 항상 "일정 승인 대기"가 되어 혼자 쓰는 사람이 자기 업무를 자기가 승인해야 함 → 일상 사용의 마찰.
- 오늘 화면의 예정 업무 행은 **행 전체가 완료 토글 버튼**이라 모바일에서 실수로 탭하면 업무가 즉시 완료(100%) 처리됨.
- 프런트 테스트 53개는 전부 통과. CI 구성(빌드+테스트+컨테이너)은 정상.

# Wave 3 — 코드 감사로 찾은 실사용 버그 (완료)

최종 갱신: 2026-07-10

Wave 2까지 커밋된 상태에서 새 기능 대신 정확성 감사를 진행(백엔드 위험 모듈 정밀 감사 + 프런트 화면 전수 점검). 아래는 실제로 재현 가능한 버그로 확인해 고친 것만 기록.

22. **`Today.jsx`의 "오늘" 날짜가 페이지 로드 시점에 고정됨**: `now`/`dateText`가 모듈 최상단 상수라 브라우저 탭을 자정 너머로 켜 두면(이 앱은 PWA로 계속 켜 두는 용도) "오늘 일정" 필터와 헤더 날짜가 영원히 어제 날짜로 고정됨. 컴포넌트 본문으로 이동해 렌더마다 재계산하도록 수정(커밋 `fe29afa`).
23. **`PATCH /api/tasks/{id}`·`PATCH /api/events/{id}`가 빈 body를 그냥 통과시킴**: `update_item()`이 `updated_at` 갱신값을 먼저 채운 뒤에 "변경 없음" 검사를 해서, tasks/events는 검사가 죽은 코드였음(todos/work_logs는 정상 동작). 검사 순서를 바꿔 네 테이블 모두 동일하게 422를 반환하도록 수정(커밋 `cafc6e5`). `POST /api/ai/apply`가 빈 데이터로 조용히 no-op하는 경로도 함께 막힘.
24. **사이드바가 로그인한 사용자의 실제 이름을 표시하지 않음**: `AppShell.jsx`가 `user?.name`/`user?.user_id`를 읽었지만 `/api/auth/me`는 `{id, email, display_name, picture_url}`을 반환 — 항상 "나의 작업 공간" 기본값과 이메일 기반 이니셜만 보였음. `display_name`/`id`로 수정(커밋 `f6f3387`).
25. **헤더의 종 아이콘 버튼이 아무 동작도 하지 않음**: 앱의 다른 모든 아이콘 전용 버튼과 달리 `onClick`도 `aria-label`도 없는 죽은 UI. 제거(커밋 `264bf51`).
26. **설정 화면 AI 섹션 인코딩 깨짐이 실제로는 안 고쳐져 있었음**: Wave 1 P0 item 1이 "완료"로 기록했지만 프런트 절반은 실제로 미반영. 재점검으로 발견해 실제로 복구(커밋 `adb3e02`). 위 P0 item 1 참고.
27. **CI가 프런트 유닛 테스트를 실행하지 않음**: `frontend` job이 `npm run build`만 실행하고 `npm test`(54개 테스트)는 건너뜀 — 순수 로직 모듈 회귀가 CI로 잡히지 않음. 테스트 스텝 추가(커밋 `c91d030`).
28. **AI 로컬 규칙 파서가 "1월 5일"처럼 연도 없는 날짜를 항상 올해로 해석**: 연말에 내년 날짜를 말하면 이미 지난 날짜가 조용히 만들어짐. 계산한 날짜가 오늘보다 이전이면 내년으로 굴리도록 수정(커밋 `ab27991`).

진행 방식 메모: 이번 웨이브는 새 audit fork가 두 번 잘못된 판단(이미 있는 기능을 "없다"고 보고)을 한 뒤라, 이번엔 fork 보고를 그대로 믿지 않고 직접 파일을 읽어 재현 조건을 확인한 것만 반영. `update_item` 버그(23번)는 fork가 찾았지만 나머지는 전부 직접 읽으며 발견.

# Wave 4 — 사용자 요청 기능 (완료)

최종 갱신: 2026-07-10

Wave 3 정적 감사 이후, 실제 헤드리스 브라우저(Playwright + 로그인 우회 세션 토큰)로 화면을 직접 띄워 스크린샷으로 검증하는 절차를 처음 도입. 사용자가 새 기능도 제안해 달라고 해서 후보를 냈고, 사용자가 두 개를 골랐다.

29. **삭제 실행 취소 토스트**: 업무·일정·오늘 할 일·업무 기록을 삭제하면 뜨는 토스트에 "실행 취소" 버튼 추가. 이미 있는 휴지통 복원 API(`api.restoreTrash`)를 그대로 재사용. `notify()`/`mutate()`/`askDelete()`에 선택적 `undo` 콜백을 추가하는 방식으로 네 개 삭제 지점 전부에 한 번에 적용(커밋 `cf70a7c`).
29-2. **Ctrl/⌘+K 빠른 입력**: 어느 화면에서든 단축키(또는 사이드바 "빠른 입력" 버튼)로 모달을 열어 자연어로 업무·일정·할 일·기록을 즉시 생성. 새 백엔드 엔드포인트 없이 AI 도우미 화면과 같은 `api.aiPreview`/`api.aiApply` 미리보기→적용 파이프라인을 재사용(커밋 `47dd36c`). 헤드리스 브라우저로 Ctrl+K → 자연어 입력 → 미리보기 → 적용 → 캘린더에 실제 반영까지 전 과정 확인.
30. **(브라우저 검증 중 추가 발견) `.done-notes` 글자가 세로로 한 자씩 쪼개져 보임**: `.done-notes div`(자손 선택자)가 각 기록 행뿐 아니라 그 안에 중첩된 내용 `<div>`에도 매치되어, `grid-template-columns:24px 1fr auto`가 텍스트 span에 그대로 적용됨 — 24px짜리 첫 칸에 억지로 들어간 글자가 한 자씩 줄바꿈됨. `.done-notes>div`(직계 자식만)로 범위를 좁혀 해결(커밋 `cf70a7c`, `9ebfd23`의 요약-여백 수정과 마찬가지로 정적 코드 리뷰로는 못 찾고 실제 화면을 봐야만 드러난 버그).
